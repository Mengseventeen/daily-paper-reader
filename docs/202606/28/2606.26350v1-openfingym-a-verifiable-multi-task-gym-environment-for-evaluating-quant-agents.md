## 摘要
尽管大语言模型智能体越来越多地应用于量化金融工作流，但其评估仍然分散在孤立的任务上，而基准任务的金融相关性往往被忽视。然而，金融工作流本质上是多阶段的，涵盖预测、策略构建、风险管理和交易等相互依赖的任务。现有平台通常集中于单一任务，因此可能夸大智能体的能力，未能揭示其在泛化、真实市场交互和具有金融意义的决策方面的弱点。我们提出了OpenFinGym，一个统一的Gym环境，用于量化金融智能体开发，在单一执行和验证接口下涵盖预测、市场生成、实时交易和欺诈检测。OpenFinGym还提供了一个自动任务构建流水线，将量化金融出版物转化为可执行任务包；一个容器化运行时，带有主机端验证器服务，支持可扩展的智能体部署，并防止运行时训练-测试泄露；一个具有低延迟数据流设计的模拟交易引擎；对长期和事件市场预测的延迟解决支持；以及用于监督微调和强化学习后训练的集成。

## Abstract
Although large language model agents are increasingly applied to quantitative-finance workflows, their evaluation remains fragmented across isolated tasks, while the financial relevance of benchmark tasks is often overlooked. Yet financial workflows are inherently multi-stage, spanning interdependent tasks such as forecasting, strategy construction, risk management, and trading. Existing platforms typically focus on a single task, and can therefore overstate agent competence and fail to reveal weaknesses in generalization, real-market interaction, and financially meaningful decision-making. We introduce OpenFinGym, a unified gym environment for quantitative-finance agent development that covers forecasting, market generation, real-time trading, and fraud detection under a single execution and verification interface. OpenFinGym additionally provides an automated task-construction pipeline that turns quantitative finance publications into executable task packages; a containerised runtime with a host-side verifier service that supports scalable agent rollouts and prevents runtime train-test leakage; a paper trading engine with a low-latency data-stream design; deferred-resolution support for long-horizon and event-market forecasts; and integration for SFT and RL post-training

# OpenFinGym: 论文总结

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：大语言模型智能体（LLM agents）在量化金融领域的应用日益增多，但现有评估体系存在碎片化问题。金融工作流本质上是多阶段的，包含预测、策略构建、风险管理和交易等相互依赖的任务。
- **问题**：现有平台（如FinRL-Meta、TradeMaster、FinBen等）通常只关注单一任务（如仅交易或仅预测），导致：
  - 夸大agent能力：无法评估agent在多任务间的泛化能力；
  - 缺乏金融相关性：任务设定脱离真实金融工作流的衔接性；
  - 缺少统一的执行和验证接口，难以进行公平比较和防止数据泄漏。
- **目标**：构建一个统一的、可验证的多任务Gym环境，覆盖预报、市场生成、实时交易和欺诈检测，并支持可扩展的agent评估与后训练。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- **统一任务抽象**：每个任务用三元组 `(D, T, E)` 表示（数据集、任务描述、评估协议）。任务包封装agent可访问的训练数据和测试输入，但测试标签仅对主机端验证器（verifier）可见，防止运行时泄漏。
- **四种任务族**：预测（48个任务）、市场生成（13个）、交易（10个）、欺诈检测（7个），涵盖股票、外汇、大宗商品、加密货币、限价订单簿、收益率曲线等资产类别。

### 2.2 关键技术细节
- **自动任务构建管道**（四阶段）：
  1. **论文收获**（Paper harvesting）：自动筛选金融论文，提取实验设置、数据集、评估指标；
  2. **数据集与奖励构建**（Dataset & Reward construction）：通过生成器-审查器循环（LLM编写脚本 + 单元测试审查）将任务规范物化为可执行数据加载器和奖励函数；
  3. **任务组装**（Task assembly）：将数据集、评估器和任务接口封装为标准化包，包含特征泄漏检查和烟雾测试；
  4. **交易任务路由**：由于交易引擎的复杂性，交易任务通过参数化预建引擎（模拟交易引擎 + 低延迟数据流）实现。
- **容器化运行时**：
  - Agent在容器内运行，仅包含训练数据和测试输入，测试标签从容器挂载中排除；
  - 主机端验证器加载真实标签并暴露速率受限的提交API，仅返回评分，不返回标签，防止反向工程；
  - 支持批次式（一次性提交）和序列式（逐步交互）两种交互模式。
- **低延迟数据流**（实时交易）：
  - 使用WebSocket订阅持续刷新内存缓冲区，每个gym step是常数时间内存读取；
  - 与REST方案相比，延迟减半、刷新率提升一个数量级（见表3：Alpaca WS延迟54.2ms vs REST 103.1ms；刷新率13.53Hz vs 1.40Hz）。
- **延迟解析机制**（Deferred-resolution）：对长期价格预测或事件市场预测，将提交记录到SQLite账本，待结果时间到达后由解析器获取真实结果并计算奖励。
- **后训练集成**：支持监督微调（SFT）和强化学习（GRPO）。SFT使用低秩适配（LoRA），RL使用SkyRL框架，奖励映射为最大化友好形式。

## 6. 主要结论与发现

1. **无单一主导模型**：不同任务族有不同领先模型：
   - 预测：GPT-5.1-codex-mini（reward 0.036，最优）
   - 市场生成：Sonnet 4.6（reward 7.114，最低）
   - 交易：GPT-4o（reward 19.912，最高）
   - 欺诈检测：Sonnet 4.6（reward 1.630，最高）
   - 所有模型在预测、交易、欺诈检测上的成功率为100%，说明前沿LLM能可靠地产生可执行量化金融代码。

2. **后训练效果显著**：
   - 所有Qwen3基座模型（1.7B/4B/8B）在留出预测任务上成功率为0%，无法产生可执行代码；
   - SFT将成功率提升至50-100%不等（均值约60-80%），但部分任务仍失败；
   - RL将成功率全部提升至100%，且奖励（损失）比SFT阶段降低一个数量级以上（如Treasury从约1降低到0.1，Crypto从约0.5降低到0.007）。

3. **OpenFinGym具备评估差异化的能力**，能够区分agent在不同金融场景下的强弱。
