## 摘要
每只股票（per-ticker）预测模型主导着金融时间序列研究，但対跨公司传播仍然视而不见：台湾的晶圆厂中断事件在苹果自身股价变动之前，单资产模型中并无反映。为解决这一局限，我们引入一种异构的 Rust-Python 流式架构，该架构将跨公司注意力映射为一个直接从文本驱动的连续时间图。我们展示，在数据摄取端，零拷贝 Rust 边解析新闻记录约需 100 ns，扫描目标股票池约需 1.2 μs。在推理端，一个多变量神经霍克斯过程（Neural Hawkes Process）采用每节点连续时间 LSTM 状态和双线性潜在投影来传播有向激励，同时一个自适应剪枝规则限制动态邻域更新的计算成本。结合这些阶段，我们在单个商用 CPU 上实现了每到来新闻记录约 13 ms 的端到端处理延迟。在对 FNSPID 语料库（47 只股票共 638 篇文章）进行的一个月时间留出评估中，该系统在 90% 分位次日回报阈值下，精确度比随机提升 1.70 倍，比同行业基准提升 3.36 倍。至关重要的是，移除图拓扑结构后精确度降为零，这证实了动态注意力网络是该架构中跨公司信号的唯一驱动因素。

## Abstract
Per-ticker forecasting models dominate financial time-series work yet remain blind to cross-company propagation: a foundry disruption in Taiwan does not register in a single-asset model until Apple's own price has already moved. To address this limitation, we introduce a heterogeneous Rust-Python streaming architecture that maps cross-company attention as a continuous-time graph driven directly from text. We show that on the ingestion side, a zero-copy Rust edge parses news records in $\sim$100 ns and scans the target equity universe in $\sim$1.2 $μ$s. On the inference end, a multivariate Neural Hawkes Process featuring per-node continuous-time LSTM states and a bilinear latent projection propagates directed excitation, while an adaptive pruning rule bounds the computational cost of dynamic neighborhood updates. Combining these stages, we demonstrate an end-to-end processing latency of $\sim$13 ms per incoming news record on a single commodity CPU. Evaluated on a one-month temporal holdout of the FNSPID corpus (638 articles across 47 tickers), the system delivers a $1.70\times$ precision lift over random at the 90th-percentile next-day return threshold, and $3.36\times$ over a same-sector baseline. Crucially, removing the graph topology collapses precision to zero, confirming that the dynamic attention network is the sole driver of cross-company signal in this architecture.

- **整体含义**：提出一种异构 Rust-Python 流式架构，通过零拷贝解析新闻文本，构建连续时间注意力图，并利用神经霍克斯过程建模有向激励，实现跨公司传播的实时检测，为下游预测或风险系统提供补充特征。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- 将跨公司注意力建模为**连续时间图**，图边权重由双线性注意力从新闻嵌入和节点隐藏状态动态计算，节点状态由连续时间 LSTM（c-LSTM）随时间演化。
- 采用**零拷贝 Rust 摄取层**实现微秒级解析，**PyTorch 推理引擎**实现毫秒级推断，端到端延迟约 13 ms。

### 关键技术细节
- **零拷贝解析**：Rust 边解析 CSV/FIX 记录约 100 ns，扫描 47 只股票约 1.2 μs，无堆分配，返回原始缓冲区的字节切片引用。
- **时间戳与单调性**：强制单调时间戳（约 25 ns），可扩展为 RDTSC 指令（x86）或 CNTVCT_EL0（AArch64）。
- **语义聚类门**：滚动质心缓冲，余弦相似度 >0.35 的冗余头条被丢弃（约 507 ns）。
- **条件强度函数**：
  \[
  \lambda_j(t) = \phi\left( \mu_j + \sum_{i \in \mathcal{N}_t(j)} \alpha_{ij}(t_{k_i}) e^{-\delta_j (t - t_{k_i})} \right)
  \]
  其中 \(\phi=\text{softplus}\)，\(\mu_j\) 为基线率，\(\delta_j\) 为衰减率，\(\alpha_{ij}\) 为有向激励。
- **连续时间 LSTM**：节点隐藏状态在事件间按指数衰减，事件到达时进行标准 LSTM 更新。
- **双线性潜在投影**：计算有向激励 \(\alpha_{ij} = \phi\left( \mathbf{w}^\top \tanh( W_q \mathbf{h}_i + W_k \mathbf{h}_j + W_v \tilde{\mathbf{v}}_k ) \right)\)，保证 \(\alpha_{ij} \neq \alpha_{ji}\) 的**非对称性**，适应供应链方向性。
- **自适应边剪枝**：当瞬时激励 \(S_{ij}(t) = \alpha_{ij} e^{-\delta_j(t-t_{k_i})} < \epsilon_p\) 时剪枝，保证节点入度有界（Proposition 1），实现每节点 O(1) 邻域聚合。
- **最大似然训练**：优化 log-likelihood \(\mathcal{L}(\theta) = \sum_k \log \lambda_{j_k}(t_k) - \sum_j \int_0^T \lambda_j(s) ds\)，积分用中点近似。
- **图构造**：初始邻接矩阵由共现、语义相似性和回报相关性三个来源加权组合（权重可学习），最终权重为 (0.42, 0.34, 0.24)。

## 6. 论文的主要结论与发现

1. **传染检测有效性**：在 90th 分位阈值下，全模型精确度 15.1%，相对于随机基线（8.9%）提升 1.70×，相对于同行业启发式（4.5%）提升 3.36×。该提升在 75th–95th 阈值区间内稳定（1.36×–1.81×）。
2. **图拓扑是关键**：移除邻接矩阵后，所有阈值下精确度恒为零（0.000），证明图结构是跨公司信号的唯一来源；c-LSTM 状态和基线率在无图时毫无贡献。
3. **双线性与剪枝在本语料上无精度收益**：去掉双线性或剪枝，精确度几乎不变（均 0.151）。但双线性提供结构性非对称性（非对称指数 0.999），剪枝提供计算保证（有界入度），保留它们为部署做准备。
4. **延迟性能**：端到端每记录延迟约 13 ms，Rust 摄取层仅占 2.1 μs（<0.02%），瓶颈在句子嵌入（~8 ms）和推理（~5.3 ms）。
5. **经济信号**：强度加权多空组合在 7 个交易日中获得正年化收益和 57.1% 日胜率（因时间短，未正式报告夏普比）。
6. **预测领先时间**：强度尖峰领先实际极端回报平均 61.5 小时（中位数 48 小时），但因回报数据为日频，该测量受限于粒度。

## 7. 优点：方法或实验设计上的亮点

- **系统设计创新**：将零拷贝低级解析（Rust）与高灵活推理引擎（PyTorch）分离，实现微秒级摄取 + 毫秒级推理，为金融流式场景提供实用架构。
- **数学建模严谨**：连续时间 LSTM 自然处理不规则时间间隔，双线性投影捕获非对称有向激励，自适应剪枝保证计算上界（Propositions 1）。
- **图构造可学习**：邻接矩阵由三个来源加权组合，权重可联合优化，避免手工设计。
- **消除信息泄露**：严格时间留出，分位阈值从预训练数据估计，避免未来信息。
- **消融实验彻底**：逐一移除关键组件（图、双线性、剪枝），验证每个组件的必要性或冗余性，结论清晰。
- **重现性**：代码公开（GitHub），确定性设置，便于独立验证。

## 8. 不足与局限

- **实验覆盖有限**：仅使用一个月、47 只股票的数据集，未在多个月份、更多股票或不同市场环境下验证，泛化能力存疑。
- **评估粒度粗**：回报数据为日频，而新闻传播实际发生在分钟尺度；论文承认这一局限，并将“20–60 分钟”延迟作为未来工作。
- **单一随机种子**：主模型仅用一个种子（seed=42），缺乏统计稳健性检验（仅随机基线使用多个种子）。
- **基准对比不完整**：同行业基线只在 90th 分位报告，未提供其他阈值下的精确度；且未与除了随机/同行业之外更复杂的基线（如统计网络方法 Diebold-Yılmaz）对比。
- **数据量小导致消融结果平坦**：双线性、剪枝等组件在本语料上无精度影响，可能是数据稀疏性的结果，而非真正的冗余。需要更大语料验证。
- **无端到端嵌入微调**：句子编码器固定（MiniLM-L6-v2），未与 Hawkes 过程联合训练，可能限制表示特异性。
- **仅 CPU 实验**：虽然论文强调无需 GPU，但在 GPU 上可能更快；且未在更高吞吐量场景下测试压力。
- **经济信号解释力弱**：组合信号仅 7 天，统计力不足，论文本身也谨慎地未给出夏普比等风险调整指标。
- **可部署性质疑**：生产环境中 MRN FIX 流可能包含更复杂的格式和噪声，零拷贝解析的实际鲁棒性未验证。
