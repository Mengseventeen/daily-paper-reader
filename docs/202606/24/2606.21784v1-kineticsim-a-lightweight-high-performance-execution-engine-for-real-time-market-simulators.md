## 摘要
使用多智能体（基于智能体）模型大规模模拟金融市场对于市场设计、监管压力测试和强化学习至关重要，但传统的CPU模拟器受限于顺序处理，而向量化的GPU框架则受到内核启动开销和冗余全局内存往返的瓶颈。我们形式化、分析并评估了一种可重用的并行设计模式：用于迭代多智能体归约的持久化、状态承载的清算。通过将可变模拟状态缓存在跨步骤边界的线程块共享内存中，通过共享内存原子操作聚合智能体动作，并协作解析清算函数，该模式将每步关键路径深度从顺序清算的Theta(L+A)（L为价格网格刻度数，A为智能体数）降低到Theta(log L + ceil(A/L))，并使全局内存流量与步数无关。我们在KineticSim中实现了这一功能，这是一个轻量级GPU执行引擎，可并行模拟大规模限价订单簿集合，达到每秒超过547亿个智能体事件的峰值吞吐量。在固定工作负载下，它比CPU（NumPy）加速3406倍，比PyTorch GPU加速27.8倍，比JAX GPU加速42.8倍，比朴素的定制CUDA基线加速8.4倍，同时使用的GPU内存大约比PyTorch少一个数量级。在53种配置中，两个定制CUDA引擎生成比特级相同的订单簿，聚合统计信息与CPU参考值偏差在0.1%以内。该模式可推广到其他需要状态持久、块局部归约的迭代多智能体工作负载。

## Abstract
Simulating financial markets at scale with multi-agent (Agent-Based) models is critical for market design, regulatory stress-testing, and reinforcement learning, but traditional CPU simulators are bottlenecked by sequential processing while vectorized GPU frameworks suffer from kernel-launch overhead and redundant global-memory round-trips. We formalize, analyze, and evaluate a reusable parallel design pattern: persistent, state-carrying clearing for iterative multi-agent reductions. By caching mutable simulation state in thread-block shared memory across step boundaries, aggregating agent actions via shared-memory atomics, and resolving the clearing function cooperatively, the pattern reduces the per-step critical-path depth from Theta(L+A) for sequential clearing (L price-grid ticks, A agents) to Theta(log L + ceil(A/L)) and makes global-memory traffic independent of the step count. We implement this in KineticSim, a lightweight GPU execution engine that simulates massive ensembles of limit-order books in parallel, reaching a peak throughput of over 54.7 billion agent-events per second. On a fixed workload it delivers speedups of 3406x over CPU (NumPy), 27.8x over PyTorch GPU, 42.8x over JAX GPU, and 8.4x over a naive custom CUDA baseline, while using roughly an order of magnitude less GPU memory than PyTorch. Across 53 configurations the two custom CUDA engines produce bitwise-identical order books, and aggregate statistics match the CPU reference to within 0.1%. The pattern generalizes to other iterative multi-agent workloads requiring state-persistent, block-localized reductions.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **问题**：大规模多智能体金融市场模拟（如用于市场设计、监管压力测试和强化学习）面临严重的计算瓶颈。传统CPU模拟器（如ABIDES）因顺序处理而受限，无法扩展到大量并行市场；而基于向量化的GPU框架（如PyTorch、JAX）虽然试图利用GPU并行性，但存在两个关键弱点：(a) **全局内存带宽瓶颈**——每步必须在GPU全局内存中读写限价订单簿（LOB），消耗大量带宽；(b) **内核启动开销**——每步从CPU启动独立内核，对于细粒度模拟，启动延迟可能主导执行时间。
- **整体含义**：作者认为现有方法无法高效支撑大规模、精细的迭代多智能体模拟，需要一种新的并行设计模式来突破这些瓶颈。KineticSim通过提出“持久化、状态承载的清除”模式，将可变状态缓存在GPU线程块共享内存中，从而大幅降低关键路径深度和内存流量，实现接近硬件极限的吞吐量。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：设计一种可重用的并行设计模式——**持久化状态承载清除**，用于迭代多智能体归约。该模式将每个市场的限价订单簿（LOB）状态存储在CUDA线程块的共享内存中，跨步骤边界保持状态，使用共享内存原子操作聚合智能体动作，并协作执行清除（扫描、归约）函数。
- **关键技术细节**：
  - **线程与块映射**：每个市场映射到一个CUDA块，块内线程数等于价格网格等级数L（通常为128或1024的幂次）。线程t对应价格t，负责初始化、聚合和清除。
  - **共享内存驻留**：LOB数组（s_bid, s_ask等）声明为`__shared__`，在模拟全过程中不离开共享内存。对于L=128，仅需4KB共享内存，远低于现代GPU SM的100KB+预算。
  - **智能体决策与订单聚合**：每个块内的线程通过网格跨度循环处理该市场内的所有A个智能体。智能体使用**无状态计数器型随机数生成器**（SplitMix64）生成决策，避免全局内存状态存储。订单数量通过`atomicAdd`在共享内存中聚合到s_BUY和s_SELL数组。
  - **协作并行清除**：使用并行Hillis-Steele前缀/后缀扫描计算累计需求D_cum和供应S_cum，以及使用并行锦标赛归约找到清除价格p*。将清除的关键路径深度从串行的Θ(L)降至Θ(log L)。
  - **持久内核调度**：整个S步骤循环在一个内核线程块内完成，只需一次主机启动，后续步骤完全在GPU上执行，消除每步内核启动开销和全局内存往返。仅最终状态写回全局内存。
- **复杂性分析**：
  - 串行朴素核：每步深度Θ(L+A)，全局内存流量Θ(S·M·L)
  - KineticSim：每步深度Θ(log L + ⌈A/L⌉)，全局内存流量Θ(M·L)（与S无关）

## 3. 实验设计

- **数据集/场景**：使用合成的均匀价格集合竞价市场模拟。价格网格L=128，每市场智能体数A可变（16,64,256,1024），并行市场数M可变（64,256,1024,4096,16384），模拟步数S=500。此外还进行了参数扫描（智能体类型比例）。
- **Benchmark**：对比五个后端：
  - CPU (NumPy)：向量化但不使用多线程
  - PyTorch GPU：高度优化的向量化实现（`torch.no_grad()`，in-place操作，异步CUDA事件计时）

  - Naive Custom CUDA：每个市场单线程，LOB在全局内存，串行扫描/归约
  - KineticSim：提出的持久块共享内存并行清除引擎
