## 摘要
金融多模态推理需要智能体在异构证据源之间协调数值计算、检索、视觉解释和时间定位。现有的工具增强型智能体提高了执行保真度，但在不同会话之间仍然基本无状态，反复重新发现推理策略和失败模式。在高风险的金融环境中，这导致不可靠的工具路由、噪声检索和易产生幻觉的推理。我们提出了FinAcumen，这是一个以选择性经验记忆为核心的金融推理智能体框架，用于工具增强的多模态推理。FinAcumen从先前轨迹中积累金融基础推理经验，将成功策略和失败衍生的警示规则提炼到持久记忆库中。在推理时，仅当语义相关性超过校准阈值时，检索到的经验才会条件化推理，而无关记忆则通过回退机制被明确抑制。一个确定性的金融工具环境进一步为数值计算、检索、视觉解码和答案验证提供了基础。在四个金融多模态推理基准测试中，FinAcumen持续改进冻结的8B视觉语言模型，超越金融专用模型，并接近领先的专有通用模型。进一步分析表明，选择性经验激活在检索不确定性下提高了推理可靠性。我们的代码匿名公开于 https://anonymous.4open.science/r/FinAcumen

## Abstract
Financial multimodal reasoning requires agents to coordinate numerical computation, retrieval, visual interpretation, and temporal grounding across heterogeneous evidence sources. Existing tool-augmented agents improve execution fidelity, yet remain largely stateless across episodes, repeatedly rediscovering reasoning strategies and failure patterns. In high-stakes financial settings, this leads to unreliable tool routing, noisy retrieval, and hallucination-prone reasoning. We present FinAcumen, a financial reasoning agent framework centered on selective experience memory for tool-augmented multimodal reasoning. FinAcumen accumulates financially grounded reasoning experience from prior trajectories, distilling successful strategies and failure-derived cautionary rules into a persistent memory bank. During inference, retrieved experiences condition reasoning only when semantic relevance exceeds a calibrated threshold, while irrelevant memory is explicitly suppressed through a fallback mechanism. A deterministic financial tool environment further grounds numerical computation, retrieval, visual decoding, and answer verification.Across four financial multimodal reasoning benchmarks, FinAcumen consistently improves a frozen 8B vision-language model over finance-specialized models and approaches leading proprietary general-purpose models. Further analysis shows that selective experience activation improves reasoning reliability under retrieval uncertainty. Our code is anonymously available at https://anonymous.4open.science/r/FinAcumen

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：金融多模态推理任务要求智能体同时协调文本、表格、图表、时间序列等多种证据源，进行数值计算、检索、视觉解读和时间定位。现有工具增强型智能体（如ReAct）虽然提升了执行保真度，但跨回合（episode）间完全无状态，每次推理都从头开始探索策略，反复遭遇相同的失败模式。这种“无记忆”行为在高风险金融场景中引发工具路由不可靠、检索噪声大、易产生幻觉（hallucination）等问题。
- **背景**：已有金融领域基准（如BizBench、FinMMR、FinTMMBench、FinMME）暴露了不同方面的挑战；金融专用LLM（如Fin-R1、Open-FinLLMs）主要依赖参数更新注入领域知识，但静态微调难以适应多变的模态组合和检索不确定性。人类专家在金融分析中不仅依赖领域知识，还依赖累积的“经验”（何时检索、如何跨模态验证等），而现有AI系统缺乏此类跨回合经验积累机制。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- **FinAcumen** 框架围绕**选择性经验记忆**（Selective Experience Memory）构建。它允许一个冻结的视觉语言模型（VLM）通过外部记忆库积累历史推理经验，在推理时仅当检索到的经验与当前问题语义相关时注入，否则回退到基础工具增强推理，从而避免噪声信息干扰。

### 关键技术细节
1. **金融记忆（Financial Memory, FM）模块**
   - **记忆库结构**：每条记录包含原始问题、黄金答案及**蒸馏经验**——成功轨迹中提炼的**策略（Findings）** 和失败轨迹中提炼的**警示规则（Cautions）**。
   - **记忆生成（Consolidation）**：对每个训练问题，模型生成多条独立轨迹（K条），根据与黄金答案的正确性评分后，由汇总智能体（Summary Agent）从成功轨迹中提取策略、从失败轨迹中提取警示，并写入记忆库。
   - **记忆激活（Memory Activation）**：推理时，将当前查询嵌入与记忆库中各条目的嵌入计算余弦相似度，仅保留相似度超过阈值τ的条目，去重排序后取前k个注入提示。若无一超过阈值（Mx*为空），则模型直接使用工具增强推理（fallback）。
   - **阈值机制**：τ=0.65（经分析确定），在该阈值下，记忆条目仅在语义充分匹配时才激活，抑制噪声记忆注入。

2. **金融工具（Financial Tools, FT）**
   - 一个确定性执行层，包括四个工具：
     - **数值推理引擎**：通过持久命名空间执行Python代码，支持级联运算（如复利、跨报表聚合）。
     - **基础数据检索**：从OHLC行情、新闻、季度指标表中查找事实（符号+日期+指标）。
     - **视觉结构解码器**：先OCR转录图表/表格为结构化文本；推理中若检测到数值矛盾可重新解析图像。
     - **答案聚合门（Answer Consolidation Gate）**：作为唯一终止点，检查答案是否源可追溯、单位一致、格式有效。
   - 工具机制在所有配置（有/无记忆）下保持一致，保证性能差异仅来自记忆。

3. **代理架构**
   - 所有角色共享同一基座模型（冻结），通过提示区分行为：采样智能体生成多样轨迹；汇总智能体蒸馏经验；标注智能体将检索到的经验重写为当前问题的个性化指令；求解智能体规划并执行工具增强推理。

4. **算法流程（文字说明）**
   - 训练阶段：对每个训练问题生成K条轨迹→评分→汇总→写入FM。
   - 推理阶段：计算查询与记忆库的余弦相似度→筛选阈值以上条目→去重排序→取top-k→重写为指令→注入提示→执行工具增强推理→终止于答案门。
