## 摘要
LLM代理越来越多地被用作自主投资组合经理，基准测试也已从金融问答转向顺序交易。然而，大多数基准仍然在一个固定的时间窗口内根据回报对代理进行排名——这是一个弱代理，因为一个时期的回报由市场路径主导，一旦控制了前瞻性泄漏，明显的阿尔法可能会消失。这样的排名既不证明合理的推理，也不证明一致的策略，也不证明持久的优势。我们引入CLQT，它将闭环交易评估重新定义为诊断而非排名：一种定位代理过程成功或失败的位置和原因的仪器。CLQT是一个完全闭环、成本感知、策略一致、时间门控的环境，其代理运行一个五阶段循环：收集、综合、分配、执行、反思。每一轮输出一个完整的决策轮，并密封到可重体验证的哈希链中，因此每个指标都可以从跟踪中重建。六个支柱构成了基底：一个严格的时间门、机构交易和融资成本建模、策略一致性评分、三层记忆、一个模型-上下文-协议工具层，以及任务感知综合。同一代理作为专业角色的约束委员会或单一的全自主编排者运行，使得过程支架成为一个实验变量。从审计跟踪中，我们计算一个五轴能力记分卡（APM-CS：连贯性、敏锐性、沉着性、纪律性、可靠性），其中连贯性部分由一个持有、非队列的LLM评判，以抑制自我偏好偏差。我们在一个污染控制的多模型回测中验证它，包括一个消融网格和一个在未见过的、截止后数据上的实时经纪商跟踪，并对照重复运行的噪声基准。CLQT将结果与能力分离，产生的不再是模型排名，而是一个持久、可扩展的代理能力和局限性地图。

## Abstract
LLM agents are increasingly cast as autonomous portfolio managers, and benchmarks have moved from financial question-answering to sequential trading. Yet most still rank agents by returns over a fixed window -- a weak proxy, since a period's return is dominated by the market path and apparent alpha can dissolve once look-ahead leakage is controlled. Such a ranking certifies neither sound reasoning, nor a consistent strategy, nor a durable edge. We introduce CLQT, which reframes closed-loop trading evaluation as diagnosis rather than ranking: an instrument that localizes where and why an agent's process succeeds or fails. CLQT is a fully closed-loop, cost-aware, strategy-consistent, temporally-gated environment whose agents run a five-stage cycle: gather, synthesize, allocate, execute, reflect. Each round emits a complete DecisionRound sealed into a recompute-verifiable hash chain, so every metric is reconstructable from the trail. Six pillars form the substrate: a hard TimeGate, institutional transaction- and financing-cost modeling, strategy-consistency scoring, three-tier memory, a Model-Context-Protocol tool layer, and mandate-aware synthesis. The same agent runs as a constrained committee of specialized roles or a single full-autonomy orchestrator, making process scaffolding an experimental variable. From the audit trail we compute a five-axis capability scorecard (APM-CS: Coherence, Acuity, Composure, Discipline, Reliability), with Coherence judged partly by a held-out, out-of-cohort LLM to curb self-preference bias. We validate it on a contamination-controlled multi-model backtest with an ablation grid and a live broker track on unseen, post-cutoff data, against a repeated-run noise floor. CLQT separates outcome from capability, yielding not a model ranking but a durable, extensible map of agent competencies and limitations.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：现有LLM代理的闭环交易基准大多采用固定窗口内按收益率排名的“排行榜”模式。这种模式存在严重缺陷：单一时期的收益率主要受市场路径主导；一旦控制前瞻性泄漏，明显的超额收益可能消失；忽视交易成本和策略一致性；无法诊断代理推理过程是否合理。
- **整体含义**：需要一个**诊断性基准**——不是简单排名，而是定位代理在“收集→分析→决策→执行→反思”循环**何处**及**为何**成功或失败，并输出多维度能力画像。这正是CLQT的定位。

## 2. 方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- 将闭环交易评估从**排名**转向**诊断**：通过严格时间门控、真实成本建模、策略一致性测量、可验证审计链，生成五轴能力记分卡。
- 五轴记分卡（APM-CS）：
    1. **Coherence（连贯性）**：分配是否遵循自身分析？由机械信号-行动一致性和持出LLM评判两部分复合。
       - Coherence = 0.5 × 信号-行动一致性 + 0.5 × 持出评判得分（归一化[0,1]）。
    2. **Acuity（敏锐性）**：是否关注有效信号而非噪音？计算提出的交易调整方向与信息复合指标的Spearman相关性，减去与波动性干扰的相关性。
       - Acuity = ρ(Δ_prop, informative) − ρ(Δ_prop, noise)
    3. **Composure（沉着性）**：面对波动时是否过度反应？基于换手率对波动率的回归斜率与策略一致性得分均值。
       - Composure = mean(1 − overreaction_slope, consistency_score)
    4. **Discipline（纪律性）**：是否在无硬约束下自我遵守任务？包括自我约束率和剪裁依赖度。
       - Discipline = 0.5 × self_constrained_rate + 0.5 × (1 − clip_dependence)
    5. **Reliability（可靠性）**：能否妥善完成闭环？由完成率、解析失败率、截断率复合。
       - Reliability = completion_rate × (1 − parse_fail_rate) × (1 − truncation_rate)

### 关键技术细节
- **五阶段闭环**：情报获取(IA) → 多源信号合成(MSS) → 任务感知投资组合构建(MAPC) → 成本最优执行(COE) → 归因与教训提取(ALE)。
- **时间门控(TimeGate)**：所有数据访问强制点-时间标准，防止前瞻性泄漏。
- **三层记忆**：工作记忆、情节记忆（衰减α=0.95/轮）、语义记忆（衰减α=0.98/轮），每12轮合并一次。
- **哈希链审计**：每轮存储输入哈希、行动哈希及前一轮哈希，支持任意第三方验证，保证篡改可检测。
- **成本建模**：四个层级（零/低/中/高），涵盖买卖价差、佣金、滑点、融资成本，默认低层校准自实时经纪商。
- **策略一致性评分**：每轮计算Ct=0.25×(1−风格漂移)+0.25×换手一致性+0.25×信号遵守+0.25×任务对齐。
- **两种技能模式**：结构化（六个专业代理，角色分解）vs 自主（单一代理全流程决定），作为实验变量。
- **持出LLM评判**：minimax-m3（主要）和GLM-5.2（辅助），均为非队列模型，避免自我偏好偏差，使用推理模式。

## 6. 主要结论与发现

1. **能力≠收益**：能力得分领先者（deepseek结构化APM-CS 72.4）不是名义Sharpe“赢家”（qwen3结构化Sharpe 9.82实为5/26轮可靠性0.19的人为产物）。
2. **结构脚手架提升能力但幅度因模型而异**：结构化模式使所有模型的APM-CS提高（deepseek +36，gemini +6.2），但小模型在结构化中面临架构解析困难（gemini可靠性0.61）。
3. **可靠性缺陷隐藏于收益**：gemini结构化收益+24.9%但解析失败率~10/26轮；qwen3结构化因输出截断及速率限制仅完成5/26轮——收益不能揭示这些。
4. **敏锐性普遍弱**：所有可靠配置的Acuity仅在0.06–0.18，两个qwen3模式均为负值，说明提议交易追踪噪音多于有效信号。
5. **成本下可超越防御性基准，但未稳定超越指数**：可靠结构化配置平均击败6/8被动基准，但击败的主要是债券/风险平价/等权重，而非SPY；低于SPY的绝对收益源于向低回撤的风险管理权衡（最大回撤−4.1% vs SPY的−8.9%）。
6. **实时一致性差距跨赛道稳定**：信号-行动一致性 vs 持出评判差距在回测为+0.33，实时为+0.34，表明仪器在未见过数据上测量相同属性。
7. **单一牛市周期不能测试模块价值，消融总结不能按收益排序**：所有单模块和多模块去除的ΔSharpe均落在重复运行噪声界限内，但能力轴（如去掉信息簇使Acuity降至0.02）记录了真正的能力损失。

## 7. 优点

- **诊断导向**：告别单一排行榜，提供多维度能力画像，可定位失败原因。
- **完整闭环**：覆盖从数据收集到执行反思的全过程，并集成记忆、工具、成本、任务对齐。
- **哈希链审计**：确保所有实验结果可追溯、可验证，提升透明度。
- **持出LLM评判**：有效减轻自我偏好偏差，双评判确认结果稳健。
- **实时验证**：在真实经纪商平台上运行，验证仪器对外部未见数据的稳定性。
- **系统消融**：13个变体涵盖单模块、多模块簇、成本层，并反复运行以确定噪声基准，结论严谨。
- **与认知理论基础关联**：明确将设计连接到Boyd的OODA循环、Anderson的ACT*理论、行为金融等，增强可解释性。

## 8. 不足与局限

1. **单市场周期**：回测和实时均处于单一牛市（SPY+27%），模块价值（如信息堆栈、学习循环）预计在转折点或下行市场中更关键。结论不能推广到其他市轮。
2. **模型面板小**：仅5个模型（回测）+6个（实时），且实时模型与回测模型为不同世代，无法进行直接同行比较。
3. **模式混淆**：Acuity和Discipline轴因结构化与自主模式的工具分配不同而模式混淆，只能在模式内比较。
4. **Coherence轴持出评判自身局限**：主要评判minimax-m3位于实时队列中，其自身回合由自己评判（虽未发现通胀，但设计上不完全干净）。
5. **成本校准不足**：高成本层参数化导致不现实的高费用（~527 bps/轮），削弱了该层的结论。
