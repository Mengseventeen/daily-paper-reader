## 摘要
金融与表格问答不仅仅需要流畅的推理：答案必须基于支撑它们的准确事实、公式、单位、符号和尺度。一个误读的单元格或错误的操作可能悄无声息地产生看似合理但错误的结果。我们引入了 \textsc{MOCA-Agent}，这是一种声明市场代码智能体，用声明级别的验证取代了自由形式的多智能体辩论。该系统将每个问题分解为类型化的原子声明，要求专业交易者智能体买入或卖出这些声明，将其订单结算为置信度加权的接受/拒绝决策，并从市场支持的证据中合成可执行的 Python 程序。然后，一个代码感知验证器检查程序的执行、结构一致性以及常见的金融推理错误，最多进行一轮市场感知修复。在涵盖金融数值推理、通用表格推理、ESG 问答和多模态图表推理的十个公开基准测试中，\textsc{MOCA-Agent} 使用固定的 Qwen3.6-27B 骨干网络取得了强劲的性能，包括 FinQA 上的 $78.3\%$、FinanceMath 上的 $76.0\%$、MultiHiertt 上的 $71.2\%$、ESGenius 上的 $86.9\%$ 以及 FinChart-Bench 上的平均 $85.6\%$。这些结果表明，在原子声明层面聚合证据而非整个答案，可以提高高风险数值推理的鲁棒性。\footnote{代码和数据见：https://github.com/UBC-NLP/MoCA-Agent。}

## Abstract
Financial and tabular question answering requires more than fluent reasoning: answers must be grounded in the exact facts, formulas, units, signs, and scales that support them. A single misread cell or incorrect operation can silently produce a plausible but wrong result. We introduce \textsc{MOCA-Agent}, a market-of-claims code agent that replaces free-form multi-agent debate with claim-level verification. The system decomposes each question into typed atomic claims, asks specialist trader agents to buy or sell those claims, clears their orders into confidence-weighted accept/reject decisions, and synthesizes an executable Python program from market-supported evidence. A code-aware verifier then checks the program for execution, structural consistency, and common financial reasoning errors, with at most one market-aware repair round. Across ten public benchmarks spanning financial numerical reasoning, general tabular reasoning, ESG question answering, and multimodal chart reasoning, \textsc{MOCA-Agent} achieves strong performance using a fixed Qwen3.6-27B backbone, including $78.3\%$ on FinQA, $76.0\%$ on FinanceMath, $71.2\%$ on MultiHiertt, $86.9\%$ on ESGenius, and $85.6\%$ average on FinChart-Bench. These results show that aggregating evidence at the level of atomic claims, rather than whole answers, improves robustness in high-stakes numerical reasoning.\footnote{The code and data are available: https://github.com/UBC-NLP/MoCA-Agent.

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：金融与表格问答系统正被部署为自动化报告、审计和研究助手的分析骨干。然而，这类任务对错误几乎零容忍：单个误读单元格、错误公式或单位尺度偏差会悄然污染最终数字，而周围文本仍然看似合理。
- **现有方法的不足**：链式思维（CoT）、程序思维（PoT）和自由形式多智能体辩论都在**整体输出**层面操作，而非依赖的原子声明。因此存在三种典型失败模式：
  1. **静默误算**：程序语法正确但用了错误的单元格或公式，下游检查仅看输出。
  2. **不透明聚合**：多智能体辩论中，多个智能体可能流畅地犯同样错误，投票产生高置信度的错误共识。
  3. **结构性验证不足**：自优化和代码修复框架主要基于执行失败或自由形式批评，无法检测静默金融错误（如符号翻转、百分比缩放、求和行错误等）。
- **本文目标**：提出一种新的框架 MoCA-Agent，通过**声明市场**机制显式地验证每个事实、公式、单位、符号和方向，从而提升数值推理的鲁棒性和可审计性。

## 2. 论文提出的方法论

### 核心思想
- 将问题分解为**类型化的原子声明**（事实、公式、单位、符号、方向等），由多个专业交易智能体对这些声明进行买卖操作，市场结算后生成置信度加权的接受/拒绝状态，约束代码合成器仅使用市场支持的声明，最终生成可执行的 Python 程序。

### 关键技术细节

1. **Claim Catalog Builder（声明目录构建器）**  
   - 输入 `⟨q, T, C⟩`（问题、可选表格、自由形式上下文），输出类型化声明集合 `K = {km}`，每个包含 ID、种类、摘要、候选值、证据引用。声明种类包括 fact, formula, unit, sign, direction, other。同时标记问题类型 `τ`（如 percentage-change, ratio 等）。

2. **Specialist Trader Agents（专业交易智能体）**  
   - 四个角色独立对每个声明下买单（buy）或卖单（sell），每个订单包含大小 n（1-5）、价格 p（0.01-0.99）和理由。角色包括：
     - **Extractor**：聚焦事实单元格提取。
     - **Formula**：验证公式是否符合操作要求。
     - **Accountant**：检查单位和会计惯例（权重最高 1.10）。
     - **Skeptic**：主动卖出证据不足的声明（权重 0.95）。

3. **Claim Market Clearing（声明市场结算）**  
   - 对每个声明 km，计算加权买入总成交量 `Bm` 和卖出总成交量 `Sm`。  
   - 价格 `πm = Bm / (Bm + Sm + ε)`，置信度 `γm = |Bm - Sm| / (Bm + Sm + ε)`。  
   - 根据阈值判定状态：`πm ≥ 0.62` 接受，`πm ≤ 0.38` 拒绝，否则不确定。  
   - 程序级市场置信度 `Γ(U) = mean(γm)` 用于后续验证和选择。

4. **Code-Aware Verifier and Market-Aware Repair（代码感知验证器与市场感知修复）**  
   - 执行程序 P 后，验证器检查：执行成功、事实声明数量满足操作、公式/单位/符号声明存在（若需要）、未使用拒绝声明、输出格式正确、操作确实出现在源码中。每个违规扣 -0.45，满足条件奖励 +1.00 + 奖励。  
   - 合格阈值 `Φ(P) ≥ 2.20`；若不合格，执行**至多一轮**市场感知修复（携带执行错误和结构问题列表）。

5. **Hybrid Selector Committee and Conflict Arbiter（混合选择委员会与冲突仲裁者）**  
   - 当市场候选程序结构性弱（`Φ < 2.20` 或 `Γ < 0.78`），同时调用基准候选程序（无声明市场）。  
   - 四方面委员会（General, Extraction, Formula, Scale）比较两个候选，投票分裂时由冲突仲裁者撰写第三程序。

6. **多模态扩展**  
   - 对于图表基准（FinChart-Bench），仅将表格 T 替换为 VLM 对图表的一次转录，其余管线不变。

## 7. 优点

1. **方法创新**：用声明市场替代自由辩论，使每个事实的接受/拒绝状态显式化、可审计。
2. **可解释性与可审计性**：整个推理过程留下清晰的审计轨迹（哪些声明被接受/拒绝，交易员理由）。
3. **泛化能力**：在金融、表格、ESG 和图表四种模态上均表现优异，无需任务专用训练。

6. **代码感知验证**：专门针对静默金融错误（符号翻转、百分比缩放）设计结构性检查，提升鲁棒性。
7. **成本可控**：一次修复轮和至多一个额外候选，推理成本可预测。

## 8. 不足与局限

1. **成本较高**：全管线 6-10 次 LLM 调用，约 5× 单次 PoT，与一轮自由辩论相当。
2. **对声明数量敏感**：假设问题可分解为 ≤10 个原子声明；在 HiTab 这类以列归一化为主要操作而非静默误算的基准上，市场机制未恢复与微调表格专家（SS-CoT 79.10%）的差距。
3. **多模态扩展简单**：仅使用单一 VLM 转录，转录错误不经过交易员再检查，可能继承静默误读。
4. **语言和领域偏置**：所有基准为英文，角色偏置（Accountant, Skeptic）和验证器检查（百分比缩放、符号翻转）针对金融/会计惯例，未验证科学、生物医学或非英语金融语料。
5. **伦理风险**：市场置信度不应视为确定性保证，输出应经人类审查；可能存在误导性金融摘要或自动化错误主张的风险。
6. **未披露计算资源细节**：缺少 GPU 类型、数量、推理时间等具体信息，可复现性稍弱。
