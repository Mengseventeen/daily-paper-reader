## 摘要
大型语言模型（LLM）显著推动了在线数据服务的发展，尤其在金融问答（FinQA）领域。然而，这类系统仍易受数值推理幻觉的影响，这在高风险的金融应用中严重损害了可靠性。尽管检索增强生成（RAG）已被广泛采用以在外部知识中锚定响应，但它引入了三个持续挑战：噪声敏感性、计算脆弱性和可审计性危机。现有的以模型为中心的方法主要侧重于单独优化检索器或生成器，但仍难以以综合方式解决这些问题。在这项工作中，我们开创了一种以数据为中心的范式，并提出了一个新颖的框架——数据为中心的推理编译器（DCRC）。该框架通过三个连贯的阶段运行：（1）对抗性数据构建，合成带有受控噪声的训练示例以教授鲁棒性；（2）多阶段训练，培养能够进行显式证据审计和程序合成的数据为中心的结构化代理（DSA）；（3）编译并执行的推理过程，其中DSA将用户查询和检索到的文档转化为可验证、可执行的推理程序。这种数据驱动的框架通过设计确保了忠实的数值推理。我们在已有的离线基准测试上进行了大量实验，并通过在真实世界在线金融问答系统中的部署进一步验证了我们的框架。

## Abstract
Large Language Models (LLMs) have significantly advanced online data services, particularly in the domain of financial question answering (FinQA). However, such systems remain susceptible to numerical reasoning hallucinations, which critically undermine reliability in high-stakes financial applications. Although retrieval-augmented generation (RAG) has been widely adopted to ground responses in external knowledge, it introduces three persistent challenges: noise sensitivity, calculation fragility, and an auditability crisis. Existing model-centric approaches, which primarily focus on optimizing either the retriever or generator in isolation, still struggle to address these issues in an integrated manner. In this work, we pioneer a data-centric paradigm and propose a novel framework, the Data-centric Reasoning Compiler (DCRC). The framework operates through three cohesive phases: (1) adversarial data construction, which synthesizes training examples with controlled noise to teach robustness; (2) multi-stage training that cultivates a Data-centric Structuring Agent (DSA) capable of explicit evidence auditing and program synthesis; and (3) a compile-and-execute inference process, where the DSA transforms user queries and retrieved documents into verifiable, executable reasoning programs. This data-driven framework ensures faithful numerical reasoning by design. We conduct extensive experiments on established offline benchmarks and further validate our framework through deployment in a real-world online financial QA system.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：大型语言模型（LLM）在在线金融问答（FinQA）中容易产生**数值推理幻觉**，导致答案不可靠。即使采用检索增强生成（RAG）来锚定外部知识，仍面临三大挑战：**噪声敏感性**（检索到的语义相似但定义不同的数值干扰）、**计算脆弱性**（基于概率的 token 生成无法保证确定性算术运算）、**可审计性危机**（推理过程不透明，无法验证证据取舍的依据）。
- **整体含义**：现有以模型为中心的方法（分别优化检索器或生成器）无法整合解决上述问题。本文提倡**以数据为中心的范式**，通过构造高质量对抗性数据训练一个结构化代理，将查询和文档编译为可验证、可执行的推理程序，从根本上缓解数值幻觉，提升金融问答的可靠性和可审计性。

## 2. 论文提出的方法论

### 核心思想
- 提出 **Data-centric Reasoning Compiler (DCRC)** 框架，包含三个连贯阶段。
- 关键理念：**将 LLM 的不确定性限制在编译阶段（证据筛选与结构化）**，而将数值执行阶段交给确定性符号执行器，从而消除计算脆弱性。

### 关键技术细节

#### 阶段1：对抗性数据构建（Adversarial Data Construction）
- **异质共识挖掘**：使用 K 个不同模型（如 GPT-5.2 High、Gemini-3 Pro、Claude-4.5 Opus）独立生成结构化表示，通过硬投票确定正样本（多数接受且数值一致）。
- **硬负样本注入**：针对三类错误模式生成对抗性负样本：
  - **概念偏移**（Concept Shift）：语义相似但定义不同（如“净利润” vs “归属股东净利润”）。
  - **时间错配**（Time Mismatch）：同指标但年份不同。
  - **实体混淆**（Entity Mismatch）：不同主体（如子公司 vs 集团）。
- **一致性过滤**：使用验证模型检查标签与理由的语义一致性，确保数据质量。

#### 阶段2：多阶段模型训练（Multi-Stage Training）
- **结构感知监督微调（SFT）**：对关键 token（Accept/Reject/why）施加更高权重（λ=2.0），迫使模型学习分类决策边界。
- **GRPO 强化学习**：设计五维奖励函数：
  - 格式奖励（可解析的 JSON）
  - 检索 F1 奖励（证据审计准确率）
  - 一致性惩罚（拒绝的证据出现在计算逻辑中则惩罚 -5）
  - 可执行性奖励（符号执行器能正确解析）
  - 准确率奖励（最终数值匹配）
- 优化目标为组相对策略优化（GRPO），采样 G=16 个输出，通过组归一化计算优势函数。

#### 阶段3：编译与执行推理（Compile-and-Execute Inference）
- 训练好的 DSA 将用户查询和文档编译为结构化中间表示（JSON 格式），包含：
  - 子任务分解（tasks）
  - 共享证据池（pool），每个证据携带多任务决策路由（Accept/Reject 及原因）
  - 符号计算逻辑（logic），引用预注册算子库（add/subtract/divide 等）
- **确定性符号执行**：通过多跳路由聚合——符号绑定（唯一值、冲突、缺失检测）——沙箱算子执行，生成最终数值和完整审计日志。

## 3. 实验设计

### 数据集与场景
- **离线基准**：
  - FinQA（8,281 对问答，含表格-文本混合推理）
  - ConvFinQA（3,892 对话，14,115 问题，多轮推理）
- **在线场景**：部署于腾讯元宝金融问答系统进行 A/B 测试（7 天，每组 1,000 样本）。

### Benchmark 与对比方法
- **Prompting 方法**：Few-shot CoT
- **程序生成方法**：FinQANet、CBR-Ren、PoT-GPT-4
- **检索生成方法**：ArgRecog、ENCORE、APOLLO、FINDER（使用 Mistral-7B 作为生成器）
- 对照方法涵盖不同类别，并控制了模型规模（DSA-7B 与 FINDER 的 7B 参数规模匹配）。

### 主要实验结果
- DSA-7B 在 FinQA-Test 达到 **84.66%** 执行准确率，比之前最优 FINDER（75.32%）提高 9.34 个百分点。
- 在 ConvFinQA 上达到 **85.67%**，显著领先。
- 与 CoT 范式比较：DCRC 在所有规模下均优于 CoT，7B 规模提高 7.16 个百分点。
- 可审计性评价：DSA-7B 噪声过滤精度 97.2%，证据检索精度 96.4%，审计逻辑准确率 99.6%。
- 消融实验：去掉对抗性审计下降最大（7B 下降 7.1%），其余组件也有贡献。
- 超参数敏感性：标签权重 λ=2.0、正负样本比 1:2 为最优。

## 6. 论文的主要结论与发现

1. **以数据为中心的编译范式**能有效解决 RAG 下金融问答的三大挑战，显著降低幻觉率（在线从 12.44% 降至 6.82%）。
2. **结构化编译（DCRC）优于传统的 CoT 生成**，因为将不确定性限制在证据筛选阶段，数值计算由确定性执行器完成。
3. **对抗性审计训练是核心**：通过明确的拒绝理由（如 [Concept_Shift]）教会模型区分语义相似但定义不同的干扰项，这是性能提升的关键。
4. 多阶段训练（SFT + GRPO 多维奖励）协同工作，确保输出格式合规、证据审计准确、逻辑自洽且可执行。
5. 在线系统部署表明，额外 0.92s 延迟换来了 16% 的用户满意度提升，在金融高可靠场景中值得。
