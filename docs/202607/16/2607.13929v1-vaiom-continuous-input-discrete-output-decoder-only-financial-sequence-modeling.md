## 摘要
金融观测是连续、异构且嘈杂的，而仅解码器的下一个令牌模型通常围绕离散符号输入构建。我们提出了用于序数收益建模的向量输入自回归推理（VAIOM），这是一个仅解码器的Transformer，用于对一小时外汇柱线进行概率性下一个收益建模。VAIOM将输入表示与输出似然分离：连续多变量金融事件向量在输入处保留数值结构，而下一个波动率归一化收益桶上的分类分布支持交叉熵训练和似然评估。选定的0.9M混合连续输入模型结合了连续事件特征与分类资产元数据、市场状态混合收益头、缺口、波动率制度和序数辅助目标以及全序列监督。模型和预处理使用2024年之前的训练数据进行拟合；模型在2024年下半年验证集上选择，并在两个2025年测试期间无需重新拟合进行评估。在三个独立训练种子中，每个模型在两个测试半周期中都优于固定的单柱LightGBM基线。对于规范检查点，相对于LightGBM的配对增益为每个事件0.029和0.043比特。验证实验表明，在相同的分类收益目标下，连续输入优于离散令牌输入；全序列监督优于最后位置训练；辅助表示塑造与混合结构收益头一起在控制比较中提高了收益似然。一项支持性容量研究发现，评估的最小完整架构层级在当前语料库上实现了最强的验证似然。

## Abstract
Financial observations are continuous, heterogeneous, and noisy, whereas decoder-only next-token models are usually built around discrete symbolic inputs. We introduce Vector-Input Autoregressive Inference for Ordinal-Return Modeling (VAIOM), a decoder-only Transformer for probabilistic next-return modeling on one-hour foreign-exchange bars. VAIOM separates input representation from output likelihood: continuous multivariate financial-event vectors preserve numerical structure at the input, while a categorical distribution over the next volatility-normalized return bucket supports cross-entropy training and likelihood evaluation. The selected 0.9M Hybrid Continuous Input model combines continuous event features with categorical asset metadata, a Mixture-of-Market-States return head, Gap, volatility-regime, and Ordinal auxiliary objectives, and full-sequence supervision. Models and preprocessing are fit using pre-2024 Train data; models are selected on 2024H2 Validation and evaluated without refitting on two 2025 Test periods. Across three independent training seeds, every model outperforms fixed single-bar LightGBM baseline in both Test halves. For the canonical checkpoint, paired gains over LightGBM are 0.029 and 0.043 bits per event. Validation experiments show that continuous input improves over discrete-token input under the same categorical return objective, full-sequence supervision improves over last-position training, and auxiliary representation shaping together with a mixture-structured return head improves return likelihood in controlled comparisons. A supporting capacity study finds that the smallest evaluated complete architecture rung achieves the strongest Validation likelihood on the present corpus.

### 2. 论文提出的方法论：核心思想、关键技术细节

-   **核心思想**：将输入表示与输出似然解耦。
    -   **输入**：采用**连续的多变量金融事件向量**（ContIn），保留了收益、缺口、波动率等变量的原始数值关系。
    -   **输出**：预测一个**离散的分类分布**，即下一个时刻经波动率归一化后的收益落在哪个“桶”（Return Bucket）中。这使得模型能够使用交叉熵损失进行训练，并直接与概率基线模型进行比较。
-   **关键技术细节**：
    -   **输入表示**：
        -   **纯连续输入（ContIn）**：包含25个数值字段，如归一化收益、缺口、相对对数波动率、日历时间（经正余弦编码）以及5种数据质量掩码。
        -   **混合连续输入（HybridContIn）**：在ContIn基础上，加入资产标识、资产类别和时间频率等3个静态分类元数据的可学习嵌入，然后与连续投影融合。
    -   **模型架构（VAIOM骨干网络）**：
        -   采用因果解码器Transformer。
        -   **输入投影**：对连续向量使用MLP投影到128维隐藏空间。对于HybridContIn，则是先将连续向量投影，再与元数据嵌入拼接后二次投影。
        -   **位置编码**：使用可学习绝对位置编码。
        -   **Transformer块**：4层预层归一化、4头因果自注意力、前馈网络隐藏维度512。
    -   **输出头**：
        -   **独立分类头**：标准的Softmax回归。
        -   **混合市场状态头（MoMS）**：将下一个收益桶的分布表示为4个潜在预测成分的加权混合。混合权重和每个成分的分布都由网络学习得到。
    -   **训练目标**：
        -   **主要目标**：下一个收益桶的负对数似然（NLL）。
        -   **辅助目标（表示塑造）**：
            -   **缺口（Gap）辅助**：预测下一个开盘缺口所在的桶。
            -   **波动率制度（VolReg）辅助**：预测下一个时刻的波动率制度（分为5类）。
            -   **序数（Ordinal）辅助**：利用收益桶的有序性，将其转化为K-1个二分类的交叉熵损失之和。
    -   **监督范围**：
        -   **全序列监督（FullSeq）**：对序列中每个有效位置都进行下一个收益桶的预测和损失计算，增加监督信号密度。

### 8. 不足与局限

-   **数据与场景局限**：
    -   所有实验均基于**单一数据源（Dukascopy）的1小时外汇和现货金属**数据。模型在股票、期货、加密货币等其他资产类别或其他时间频率上的泛化能力尚待验证。
    -   只研究了**单个时间频率（1H）**。论文提到的4H和Daily实验是开发过程中的辅助，且表现不稳定。
-   **基线对比的不彻底性**：
    -   如前所述，LightGBM基线**未使用滞后特征或滑动窗口聚合**，这使得LightGBM在比较中处于不利地位。一个更强的、带有上下文聚合能力的LightGBM基线可能会缩小差距。
    -   缺乏与其他金融序列模型（如StockGPT, Kronos）的直接比较。
-   **输入表示的消融不够精确**：纯连续输入（ContIn）和离散输入（DiscIn）的对比并非字段严格匹配，因为ContIn缺少DiscIn中的元数据字段，所以不能将性能差异完全归因于“数据连续性”。
-   **概率提升不等于交易盈利**：论文明确指出，似然压缩（信息论指标）的提升**并不等同于**可盈利的交易策略。从预测分布到交易决策需要额外的步骤（如头寸映射、考虑成本、风险管理）。
-   **模型经济可解释性不足**：混合市场状态（MoMS）头的潜在成分被假设为**不具有经济上的可识别性**，其价值仅在似然提升上，这使得模型的内部决策过程难以解释。
-   **单点实验**：虽然做了随机种子和多层次消融，但每个消融实验和容量分析都只基于**一个随机种子**，这可能引入偶然性。多随机种子验证主要集中在最终的主模型上。
