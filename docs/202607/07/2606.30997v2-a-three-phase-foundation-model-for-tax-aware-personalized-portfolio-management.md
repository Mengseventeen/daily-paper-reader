## 摘要
我们提出了一种用于个性化投资组合管理的三阶段深度强化学习系统，解决了以往所有金融强化学习工作共有的三个局限性：1）股票代码锁定，2）单一目标，以及3）静态用户模型。第一阶段通过在多资产语料库上进行自监督学习，预训练了一个与股票代码无关的跨资产编码器，并使用基于T5的时间序列基础模型Chronos通过冻结并行分支增强，通过学习的门控机制融合。据我们所知，这是时间序列基础模型首次应用于投资组合管理强化学习。该编码器通过50维可观察元数据向量泛化到任何公开交易资产，无需为新股票代码重新训练。第二阶段使用PPO算法，在目标条件奖励下微调MoE（混合专家）投资组合行动者-评论家，该奖励同时服务于每轮采样的六个不同投资目标：短期阿尔法、短期收益、长期收益、资本保值、税收亏损收割和仅长期收益。MoE架构将每个目标分配给专门的专家头（动量、增长、防御、税收感知），并通过学习到的意图路由器根据当前目标和市场状态混合专家，消除了跨目标梯度冲突。第三阶段添加了一个轻量级个性化层，通过在实际经纪交易历史数据上微调的76参数LoRA模块，在推理时进一步适应每个个体，从揭示的交易行为中推断投资目标，而非通过问卷调查。自然语言意图解析器将自由形式的目标直接转换为结构化的投资目标参数。

## Abstract
We present a three-phase deep reinforcement learning system for personalized portfolio management that addresses three limitations shared by all prior financial RL work: 1) ticker lock-in, 2) monolithic objectives , and 3) static user models. Phase 1 pretrains a ticker-identity-free cross asset encoder via self-supervised learning on a multi-asset corpus, augmented by a frozen parallel branch using Chronos, a T5-based time series foundation model, fused via a learned gating mechanism. To our knowledge, this is the first application of a time series foundation model to portfolio management RL. The encoder generalizes to any publicly traded asset via a 50-dimensional observable metadata vector that requires no retraining for new tickers. Phase 2 fine-tunes a MoE (Mixture of Experts) portfolio actor critic with PPO under an objective-conditioned reward that simultaneously serves six distinct investment goals sampled per episode: short-term alpha, short-term gain, long-term gain, capital preservation, tax-loss harvesting, and long-term-gains-only. A MoE architecture assigns each objective to a specialized expert head (momentum, growth, defensive, tax-aware), and a learned intent router blends experts based on the active objective and current market regime, which eliminates cross-objective gradient conflict. Phase 3 adds a lightweight personalization layer further adapted at inference time to each individual via a 76-parameter LoRA module fine-tuned on real brokerage transaction history, inferring investment objectives from revealed trading behavior rather than questionnaires. A natural language intent parser converts free-form goals directly into structured investment objective parameters.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题定义**：现有金融强化学习（RL）系统存在三大局限：
  - **股票代码锁定（ticker lock-in）**：模型只能在固定资产集合上训练，无法泛化到新股票，需重新训练。
  - **单一目标优化（monolithic objectives）**：大多数方法使用单一 Sharpe 比率奖励，无法同时服务不同用户的多样化投资目标（如短期套利、长期增长、资本保值、税务优化等）。
  - **静态用户模型（static user models）**：用户偏好仅在初期通过问卷收集一次，之后不再更新，无法反映真实交易行为的变化。
- **市场空白**：现有工具分为两类——机构级高频交易系统（成本高、需要专业编程、不关注税务和个人目标）和零售级工具（基于规则、缺乏市场信号和个性化）。两者之间缺乏一个同时具备**学习到的市场信号**、**税务意识**和**个性化**的系统。
- **本文目标**：构建一个三阶段基础模型系统，实现股票代码无关、多目标联合优化、动态个性化，并首次将时间序列基础模型（Chronos）应用于投资组合管理 RL。

## 2. 方法论：核心思想、关键技术细节

### 2.1 整体架构（三阶段管道）

- **Phase 1：自监督编码器预训练（SSL Pretraining）**
  - 使用跨资产编码器（CrossAsset Encoder），输入价格窗口和50维元数据，输出每个股票的表示。
  - 融合两条分支：一个 SSL 训练的 MarketEncoder（交易特征）和一个**冻结的 Chronos 编码器**（T5 时间序列基础模型，预训练于100B+数据点），通过可学习的门控机制进行融合。
  - 元数据向量（50维）：包括行业 one-hot、市值、基本面、分析师共识、期权信号、财报日历、技术面、内部人交易、机构持仓等，可实现新股票零样本推理。
  - **互股票对比损失（Inter-ticker Contrastive Loss）**：解决表示坍缩问题（原始 SSL 导致所有股票 cosine 相似度0.96，无法区分），将相似度降至0.24。
  - 损失函数组合：`L_P1 = 0.3 L_ret + 1.0 L_mask + 0.5 L_reg + 0.5 L_contrast`

- **Phase 2：目标条件组合 RL 微调（PPO + MoE）**
  - 使用 PPO 训练一个 MoE 投资组合行动者-评论家。
  - **目标条件奖励（Objective-Conditioned Reward）**：每轮随机采样一个目标（共6种：短期 alpha、短期收益、长期收益、资本保值、税务亏损收割、仅长期收益），奖励函数根据目标不同形状不同（例如资本保值惩罚回撤，税务收割奖励 Q4 损失实现等）。
  - **4个专家头（Expert Heads）**：动量（短期）、增长（长期）、防御（资本保值）、税务感知。每个专家只从指定目标接收梯度，消除梯度冲突。
  - **意图路由器（Intent Router）**：基于市场状态和当前目标，输出混合权重融合专家输出。路由器同时会考虑市场状态（如高波动时会将部分短期目标路由到防御专家）。
  - **可学习现金令牌（Learnable Cash Token）**：强制显式现金分配，避免模型通过默认 HOLD 堆积现金。
  - **再部署奖励（Redeployment Reward）**：激励“卖出→立即买入”完整周期，避免资金闲置。
  - **硬周转上限（Hard Turnover Cap）**：防止策略发现退化的高频交易模式。
  - **六阶段课程学习（Curriculum Learning）**：分别训练单一专家、联合训练路由器、最后进行专家嫁接（Expert Grafting）恢复性能。

- **Phase 3：个性化层（LoRA 适配）**
  - 使用76参数的 LoRA 模块，仅微调意图路由器权重，专家和编码器冻结。
  - 从用户经纪交易历史中提取“揭示偏好”（如持仓时间、LT售出比例、损失收割分数等），更新用户行为向量。
  - 自然语言意图解析器：将“3年后买房”、“孩子10岁大学基金”等文本直接转换为结构化目标参数。
  - 信任优先 UX：先预览推断的偏好，用户确认后再应用。

### 2.2 关键技术细节总结

- 使用 Chronos-T5-Small（46M参数）作为冻结特征提取器，仅训练投影层（74K参数）和门控（8K参数），训练前预计算并缓存 Chronos 嵌入。
- 使用元数据替代股票代码嵌入，实现零样本泛化。
- 对比损失从0.96相似度降至0.24，雪崩效应使分配头能产生差异化权重。
- MoE 通过目标独热向量和市场状态进行路由，使用监督交叉熵损失加速路由器收敛。
- 6阶段课程：专家单独训练→路由器训练→联合训练→专家嫁接，避免了直接联合训练导致的崩溃。

## 6. 主要结论与发现

- **表示坍缩问题可解决**：标准 SSL 目标无法区分不同股票，互股票对比损失可将 cosine 相似度从0.96降至0.24，使分配头产生差异化权重。
- **MoE 消除梯度冲突**：直接联合训练6个目标导致模型快速崩溃为均匀权重；通过6阶段 MoE 课程（专家单独训练→路由训练→嫁接）实现每个专家取得正 alpha（+3.16%~+3.37%），嫁接后恢复最佳性能。
- **路由器训练需谨慎**：冻结专家训练路由器导致平坦损失面（路由权重恒为0.25）；加入监督路由损失、对角初始化、意图投影后，路由差异化显著（std>0.37）。
- **首次将时间序列基础模型（Chronos）用于组合管理 RL**：门控融合可有效结合通用时间模式与金融专用表示。
- **性能结果**：14天窗口 MoE 嫁接策略 alpha vs EW = +2.93%，优于单头策略的+2.90%；日胜率提升至42.9% vs 35.7%。
- **个性化方案可行**：76参数 LoRA 模块可在 CPU 上秒级适应，但尚未实证验证。
