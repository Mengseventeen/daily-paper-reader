## 摘要
评估大型语言模型（LLM）智能体是否能在资本市场盈利，越来越倾向于端到端交易框架：将智能体置于历史市场中，让其交易并衡量投资组合收益。但这种设置存在两个评估缺陷。首先，长时间的回测往往与前沿LLM的知识截止日期重叠，使得记忆中的股票代码、日期、价格和市场叙事替代了投资推理。其次，原始收益作为选股能力的噪声代理，因为正收益可能来自市场贝塔、风格暴露或有利的市场环境，而非真正的阿尔法。我们提出KTD-Fin（知行金融基准），一个端到端的股票市场交易基准，解决了这两个问题。KTD-Fin采用数据端掩码协议，在提示和工具中一致地匿名化关键标识符和日历信息，将历史市场记忆与投资决策分离。它还引入了Barra风格业绩归因框架，将投资组合收益分解为市场、风格和选股阿尔法成分。在2024-2026年窗口期的中国沪深300指数上评估了十个前沿LLM智能体，掩码显著改变了智能体的推理依据，使其转向匿名化的因子推理。归因分析进一步表明，在泄漏控制评估下，LLM智能体的累计收益主要由被动市场和风格暴露解释，而持续选股阿尔法的证据有限。这些发现表明，金融LLM基准不仅应评估智能体是否赚钱，还应评估收益来源是否反映了可迁移的投资技能。我们将KTD-Fin作为LLM交易智能体泄漏控制和归因感知评估的可复现模板发布。

## Abstract
Evaluating whether large language model (LLM) agents can profit in capital markets is increasingly framed as end-to-end trading: place an agent in a historical market, let it trade, and measure portfolio returns. This setup is vulnerable to two evaluation failures. First, long backtests often overlap with the knowledge cutoffs of frontier LLMs, allowing memorized tickers, dates, prices, and market narratives to substitute for investment reasoning. Second, raw returns are a noisy proxy for stock-selection ability, since positive performance may come from market beta, style exposure, or favorable regimes rather than genuine alpha.   We introduce KTD-Fin (Knowing-To-Doing Financial Benchmark), an end-to-end stock-market trading benchmark that addresses both issues. KTD-Fin uses a data-side masking protocol to anonymize key identifiers and calendar information consistently across prompts and tools, separating historical market memory from investment decision-making. It also incorporates a Barra-style performance attribution framework that decomposes portfolio returns into market, style, and stock-selection alpha components.   Across ten frontier LLM agents evaluated on the Chinese CSI300 over a 2024--2026 window, masking substantially changes agent rationales, pushing them towards anonymized factor-based reasoning. Attribution analysis further shows that LLM agents' cumulative returns under leakage-controlled evaluation are largely explained by passive market and style exposure, with limited evidence of persistent stock-selection alpha. These findings suggest that financial LLM benchmarks should evaluate not only whether an agent makes money, but also whether the source of returns reflects transferable investment skill. We release KTD-Fin as a reproducible template for leakage-controlled and attribution-aware evaluation of LLM trading agents.

## 1. 核心问题与整体含义（研究动机和背景）

- **两个关键评估缺陷**：
  - **数据泄露（Leakage）**：现代 LLM 训练数据包含大量金融信息（股票代码、价格、日期、公司叙事等），当端到端回测窗口与模型知识截止日期重叠时，智能体可能依赖记忆而非实际观测数据进行决策，导致“假性盈利”。
  - **回报噪声（Return Noise）**：原始累计回报无法区分收益来源——正收益可能来自市场贝塔、风格暴露或有利的市场环境，而非真正的选股能力（alpha）。现有基准缺乏对收益源的分解，无法衡量模型技能。
- **现有解决方案不足**：DeepFund 仅限制评估日期在模型训练截止之后（窗口缩短且无法跨模型共享）；OracleProto 仅用于静态预测且提示层限制失效。
- **核心目标**：构建一个能同时解决数据泄露和回报分解问题的端到端 LLM 股票交易基准，区分“知道”和“做到”。

## 2. 方法论

### 2.1 核心思想
- **数据端掩码协议**：在提示与工具中一致地匿名化股票代码、日历日期和时间戳，使模型无法利用预训练记忆，仅依赖数值特征进行决策。
- **Barra 风格归因框架**：每日截面回归将组合收益分解为市场（Common）、风格（Style）和选股阿尔法（Selection Alpha），分离技能与被动因子暴露。
- **三种决策模式**：memory-only（仅凭记忆）、fixed-candidate（给定因子表）、open-research（工具自主检索），分别探测记忆依赖、信号选取和自主推理能力。

### 2.2 关键技术细节
- **执行引擎**：基于 Qlib 实现 A 股真实交易约束（T+1、限价、费用等）；仅提供 OHLCV 价格数据，排除新闻/基本面，确保任何品牌或叙事推理必来自预训练记忆。
- **四层掩码**：bright（真实标识）、stock-blind（匿名股票，真实日期）、date-blind（真实股票，匿名日期）、blinded（两者均匿名）。别名独立攻击认证：联合恢复率 ≤1.5%。
- **Barra 归因公式**：
  \[
  r_{i,t} = f_{0,t} + \sum_{k} x_{i,k,t-1} \lambda_{k,t} + \varepsilon_{i,t}
  \]
  - 聚合组合收益：\( R_{pt} = f_{0,t} + \sum_k (\sum_i w_{i,t-1} x_{i,k,t-1}) \lambda_{k,t} + \alpha_{pt} \)
  - 使用 9 个 VIF 筛选的风格因子（动量、波动、流动性等）。
- **指标面板**：10 维指标涵盖收益/风险、行为（换手率、HHI、现金比例、弃权率）、可靠性（解析失败率、期望校准误差）。

## 6. 主要结论与发现

- **掩码改变交易行为**：在 bright 条件下，模型推理依赖于品牌、行业和历史事件（如“茅台，白酒龙头”）；在 blinded 下，推理转为因子排名（“20日收益率+27.4%，低波动0.0255”）。memory-only 模式下 bright 产生 -0.16% 收益而 blinded 完全持有现金（收益 0.00%）。
- **累计回报主要由被动暴露解释**：Barra 分解显示，市场成分贡献11%-29%，风格成分贡献 16%-29%，而选股阿尔法除 Claude Opus-4.7（+0.2%）外均为负值（最大 -77.8%），且跨模型方差 78pp 远超其他两项。总收益排名与选股阿尔法排名严重背离。
- **因子偏好差异**：传统模型偏好低波动、高动量的大型股；LLM 模型偏好高波动、近期活跃股（动量多空差异显著）。
- **评估应关注收益来源**：仅凭总收益无法区分技能与运气，必须分解以衡量真实选股能力。

## 7. 优点

- **创新的掩码协议**：首次在端到端交易基准中通过数据端匿名化控制预训练记忆泄漏，并附加独立攻击认证；两级中间掩码允许隔离股票和日历隐私。
- **归因框架**：引入 Barra 分解到 LLM 交易评估，分离市场、风格和选股 alpha，避免了单纯依赖总收益的误导。
- **多维度指标面板**：包括行为、可靠性、校准等多维指标，而非单一收益率。
- **复现性强**：提供了环境、脚本、掩码协议、探针代码和完整评估数据集的公开释放，可作为模板推广到其他领域。
- **实验设计严谨**：包含锚点模型深度扫描、跨模型对比、掩码验证、多种子中位数及统计检验，减少随机偏差。

## 8. 不足与局限

- **市场覆盖窄**：仅限中国 A 股沪深300，未覆盖美股、港股等，也未纳入其他资产类别（外汇、债券）。

- **锚点模型深度扫描不完整**：全掩码×模式网格仅在 Step-3.5-Flash 上完成，其他 9 个模型只在核心条件上评估，缺乏跨模型的模式效应比较。
